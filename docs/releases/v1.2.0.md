# v1.2.0 — Auditoria 10-agentes (2ª passada): 50 achados em 4 ondas

## O que mudou

A v1.2.0 fecha o **segundo ciclo de auditoria 10-agentes** em uma semana. 50 achados endereçados, distribuídos em 4 ondas temáticas: segurança (11), UX e coerência (14), regras BR saindo da doutrina (12) e qualidade interna (13). O framework ganha 5 hooks novos, README finalmente sem jargão pro não-programador, e FISCAL-004 + PIX-001/003 viram bloqueio mecânico — antes eram só doutrina.

## Por que importa

- **Pro dono de produto que não programa:** README e Quickstart agora têm trilha dupla (`QUICKSTART-DONO.md` sem jargão + `QUICKSTART-DEV.md` técnico). `ABRIR-TERMINAL.md` cobre o passo zero. Mensagens do instalador traduzidas: "IDE/CLI detectada" virou "assistente de IA detectado".
- **Pra fintech BR:** dupla cobrança Pix (a falha #1 de fintech) agora é bloqueada — hook `pix-idempotency-check.js` no addon `fintech-br` exige combinar 2 das 4 camadas (Idempotency-Key, TxId determinístico, UNIQUE no banco, lock distribuído). Matching por nome+valor é proibido — força uso de EndToEndId.
- **Pra quem emite NF-e:** declarar plano de contingência (SVC-AN/SVC-RS/EPEC) virou obrigatório. Quando SEFAZ cai, operação não trava.
- **Pra projeto que manuseia LGPD:** 3 lembretes novos — minimização, transferência internacional, DPO/canal do titular. Não bloqueiam (são doutrinários por natureza), mas avisam no momento certo, sem o auditor humano ter que lembrar.
- **Pra time SRE:** /hotfix obrigatoriamente leva a /incident-postmortem em 48h. Antes era só doutrina; agora o próximo commit não-emergencial é barrado se faltar o postmortem.

## Como aplicar

1. **Atualize:** `npx roldao-method@latest update`
2. **Verifique:** `npx roldao-method doctor`
3. **Se algo der errado:** `npx roldao-method rollback` volta pra v1.1.0.

`update` preserva todas as suas customizações (`.specify/overrides/`, `AGENTS.md`, `CLAUDE.md`, `CLAUDE.local.md`, `REGRAS-INEGOCIAVEIS.md`, `settings.local.json`, `.mcp.json`).

## Atenção

- **Nenhuma quebra de compatibilidade** com v1.1.0. Todos os hooks, skills, agentes e workflows continuam funcionando.
- **Novos overrides conscientes via variável de ambiente** (use só se realmente precisar):
  - `ROLDAO_METHOD_MCP_ALLOW_UNKNOWN=1` libera MCP fora da allowlist.
  - `ROLDAO_METHOD_FORMAT_ALLOW_GLOBAL=1` libera prettier/eslint do PATH global.
- **Hook `block-destructive.js` agora protege markers de pipeline:** qualquer `rm` em `.claude/.runtime/feature-active|auditor-*-pass|checkpoint-done|bug-active|investigator-invoked|investigation-*.json|quick-dev-files` bloqueia. Se você apagava manualmente esses arquivos pra "destravar", agora rode o agente da etapa pendente.
- **`/prd` parou de decompor stories** — fluxo unificado: `/prd` (plano) → `/epico` (decomposição) → `/historia` (1 story) ou `/feature` (executar).
- **`mascarar-dado-pessoal` em texto livre:** regex de 11/14 dígitos removida do auto-mask. CPF/CNPJ cru agora exige chamar `mascarar_cpf` / `mascarar_cnpj` direto. Auto-mask só pega formato canônico (`000.000.000-00`).
