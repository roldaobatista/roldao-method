# v1.0.0-rc1

> **Release candidate da v1.0.0.** Se ninguém abrir issue grave em 2 semanas, corta a `v1.0.0` final. Use em produção com a cautela usual de RC.

## O que mudou

O coração técnico do framework foi reescrito de bash/perl pra Node.js puro. Os 26 hooks bloqueadores agora rodam em qualquer ambiente com Node 18+ — **incluindo Windows com PowerShell ou CMD, sem precisar instalar Git for Windows**.

## Por que importa

- **Dev BR Windows puro passa a ter proteção real.** Antes, instalar em PowerShell deixava os hooks silenciosamente desativados (pior modo de falha possível). A v0.20.0 elevou o aviso pra banner gigante + bloqueio default. A v1.0 **estende a proteção**: hooks funcionam nesse ambiente.
- **Manutenção única.** Acabou o risco de divergência `.sh` vs `.ps1` — caminho B do PRD-001 (port PowerShell) foi explicitamente descartado por isso.
- **Suite de teste mais rápida.** Node test runner nativo, sem `_test-runner.sh` artesanal.
- **Tarball npm 15% menor** (menos código bash duplicado).
- **CI cobre 4 ambientes**: Ubuntu, macOS, Windows com bash, **Windows sem bash** (novo).

## Como aplicar

```bash
npx roldao-method@latest update
```

O `update` faz:

1. Substitui `templates/.claude/hooks/*.sh` por `*.js` (backup `.bak` automático).
2. Atualiza `settings.json` apontando pra `node hook.js` em vez de `bash hook.sh`.
3. Atualiza `statusline.sh` → `statusline.js`.

Quem usa addon (`fintech-br`, `electron-br`, etc.) precisa rodar também:

```bash
npx roldao-method add <addon>    # reinstala versão v1
```

## Atenção

### Breaking change

- **Hooks `.sh` foram removidos.** Quem tem customização local em `.sh` precisa migrar pra `.js` (ou colocar em `.specify/overrides/` que continua intocado).
- **`bash` e `perl` não são mais requisitos do framework.** Continuam úteis pra outros propósitos do projeto cliente, mas o ROLDAO em si não usa.
- **`_test-runner.sh` deletado.** Substituído por `test/hooks-node-only.test.js` (`npm test`).

### Migração detalhada

Veja [`docs/MIGRACAO-V1.md`](../MIGRACAO-V1.md) — cobre:

1. Como saber se sua instalação tem `.sh` antigos sobrando.
2. Como rodar `update` com segurança em projeto crítico.
3. O que fazer se você customizou um hook `.sh`.
4. Como reverter (rollback) se algo der errado.

### Compatibilidade

| Item | Funciona? |
|---|---|
| Claude Code, Cursor, Windsurf, Cline, Roo, Aider, Continue, Gemini CLI, Codex CLI | ✅ |
| Node 18 / 20 / 22 | ✅ |
| Linux, macOS, Windows (com OU sem Git Bash) | ✅ |
| Hooks personalizados em `.sh` direto em `templates/` | ❌ — migrar pra `.js` ou mover pra `.specify/overrides/` |
| Addons em v0.x (vendidos antes da v1.0-rc1) | ⚠️ rodar `add <addon>` pra atualizar |

## Como testar

```bash
# Suite local

# Smoke individual
npm run test:hooks-node-only   # 59 cenários cobrindo os 26 hooks

# Doctor no projeto
npx roldao-method doctor       # confirma todos os .js presentes + Node detectado
```
