# v0.16.0

## O que mudou

Quarta auditoria com 10 agentes encontrou um buraco importante: o framework prometia proteção LGPD pra chaves Pix em log, mas não tinha hook que bloqueasse. Esta versão fecha esse buraco com um bloqueador novo, uma skill que gera a comunicação oficial à ANPD em caso de incidente, e o procedimento operacional pra responder vazamento em até 72 horas (que é o prazo legal). Também ajusta uma série de pequenos drifts entre o que a documentação diz e o que está no código.

## Por que importa

- **Chave Pix em log agora é bloqueado antes de subir.** Antes era recomendação. Agora é hook que para o agente na hora.
- **Plano de incidente 72h pronto pra usar.** Quando incidente acontece, ninguém pensa em ordem. O runbook documenta passo a passo, e a skill gera o draft pra ANPD que o DPO só revisa.
- **Decisão arquitetural que toca dado pessoal agora exige checklist LGPD.** O modelo de ADR ganhou uma seção obrigatória pra base legal e RIPD.
- **Catálogo de skills visível.** Agora tem README na pasta `.claude/skills/` listando as 12 skills BR com tipo (validador/template) e gatilho.

## Como aplicar

1. Atualize com `npx roldao-method update`.
2. O hook novo entra automaticamente na próxima sessão Claude Code.
3. Se o projeto integra Pix, considere instalar o addon `lgpd-compliance` (`npx roldao-method add lgpd-compliance`) — traz a skill nova de comunicação ANPD.
4. Leia `docs/runbooks/incident-response-lgpd.md` antes que um incidente aconteça (não depois).

## Atenção

- Nenhuma quebra de compatibilidade. Todos os fluxos antigos seguem iguais.
- O hook `no-log-pix-key.sh` tem exceção via comentário `// PIX-004-exception: <razão>` pra casos legítimos (log interno com retenção curta + criptografia).
