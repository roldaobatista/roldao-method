# v1.1.0 — Auditoria round 11 + 2 especialistas + 6 validadores BR + 3 hooks

## O que mudou

Sessão de auditoria com 10 agentes paralelos identificou 20 achados (críticos/altos/médios/baixos). **Todos endereçados em uma janela só** — exceto refactor grande de `bin/install.js` (1879 linhas), conscientemente adiado pra v1.1.x com plano detalhado e marcadores no código.

O framework agora tem **17 especialistas** (entrou Bia 🤖 pra testes End-to-End e Marcos 🚨 pra incidente em produção), **19 skills BR no core** (6 novas — título de eleitor, CNH, RENAVAM, conta bancária, mascarar dado pessoal, PIS movido do addon) e **39 hooks** (3 novos: NF-e imutável após emissão, lembretes LGPD-002 esquecimento e LGPD-004 trilha de auditoria).

Os 2 hooks `block-destructive` e `secrets-scanner` ficaram mais robustos — bypass por escape e falso positivo de senha em comentário foram fechados, com 16 testes regressivos novos.

## Por que importa

- **Cobertura de dado pessoal completa:** quem desenvolve em PT-BR ganha skill nativa pra validar+mascarar CPF, CNH, título eleitoral, conta bancária, PIS, RG, IE — fim do "esqueci de mascarar e vazou no log".
- **Defesa anti-fiscalização:** UPDATE em NF-e já emitida agora bloqueia. Alterar XML autorizado é incidente garantido em fiscalização — o hook impede o erro antes do commit.
- **Pipeline de qualidade fechado:** já existia hook bloqueando ausência de E2E, mas ninguém escrevia. **Bia (qa-automation)** preenche o gap com Playwright/Cypress.
- **Primeiro respondente codificado:** quando algo quebra em produção, **Marcos (sre-on-call)** triagem em 5 min e escala pro Detetive (causa raiz), Bruno (fix) ou Lucas (infra). Aciona LGPD-006 (72h ANPD) quando dado pessoal afetado.
- **`dry-run` deixa de enganar:** mostrava `criados: 0` em diretório vazio (que receberia 176 arquivos). Agora mostra contagem real.

## Como aplicar

```bash
npx roldao-method update
```

- Customizações em `.specify/overrides/` preservadas.
- `AGENTS.md`/`CLAUDE.md`/`REGRAS-INEGOCIAVEIS.md` preservados.
- Settings local (`.claude/settings.local.json`) preservado.
- Snapshot automático antes — pode desfazer com `npx roldao-method rollback`.

Pra instalar do zero em projeto novo:

```bash
npx roldao-method install
npx roldao-method tutorial
```

## Atenção

**Nenhum breaking change.** Tudo é aditivo:
- Hooks novos são exit 0 (soft) ou exit 2 (bloqueador específico — NF-e emitida + comandos perigosos).
- Skills novas em pastas próprias — não tocam skills existentes.
- Agentes novos em arquivos próprios — pipeline `/feature` segue idêntico.
- Settings: `Bash(env)` e `Bash(printenv*)` moveram de `allow` pra `ask`. Em terminal de dev quase nunca aparece; em CI pode prompar 1x — autorizar `always allow` se necessário.
