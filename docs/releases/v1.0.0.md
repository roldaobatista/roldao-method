# v1.0.0

## O que mudou

Fechamento da v1.0 estável depois de 2 release candidates. Esta versão consolida o port Node dos hooks (entregue nas rc1/rc2) e aplica as recomendações da auditoria 10-agentes feita em 2026-05-24 — 3 ADRs novos formalizando débitos arquiteturais, 3 regras fiscais novas (NFS-e nacional, MDF-e/CT-e, split payment), addon `healthtech-br` em draft pra healthtech BR, e correções de consistência (status das histórias, contagens batendo, referências corrigidas).

## Por que importa

- **Promessas alinhadas com a realidade.** README, contagens e status das histórias agora batem 1-a-1 com o que está em disco.
- **46 regras rastreáveis** (eram 43). Reforma Tributária, transporte e NFS-e nacional agora têm ID citável em ADR/PR/commit.
- **Hook fiscal mais rigoroso.** Agora detecta `tpAmb=1` em 3 notações (atribuição, YAML/objeto, tag XML) — antes deixava passar YAML e XML.
- **Política de SemVer escrita.** Você sabe o que pode quebrar entre versões e o que não pode (ADR-016).
- **Python declarado.** Se você não tem Python instalado, sabe ANTES de instalar que 9 skills BR não vão validar localmente (ADR-018).
- **healthtech-br abre a porta** pra projetos brasileiros de saúde — está em DRAFT pra você revisar com seu advogado antes de usar.

## Como aplicar

Você já estava na rc2? Roda `npx roldao-method@latest update` no seu projeto e ele puxa as 3 regras fiscais novas, os 3 ADRs e a correção do hook fiscal. Suas histórias e ADRs ficam intactos.

Você nunca instalou? `npx roldao-method install` na raiz do seu projeto.

Pra instalar o addon `healthtech-br` (DRAFT — atenção):

```bash
npx roldao-method add healthtech-br
```

## Atenção

- **`healthtech-br` é DRAFT.** Não use em produção com dado real de paciente sem validar com advogado especializado em saúde/LGPD. O texto das regras `HEALTH-EXT-001..006` é base inicial — versões CFM/ANS podem ter atualizado.
- **Python 3.10+.** Se você roda em Windows puro sem Python, 9 skills BR de validação (CPF/CNPJ, Pix, boleto, NF-e, CEP, IE, IBGE) não funcionam. Instale Python (`winget install Python.Python.3.10`) ou aceite que essas validações vão acontecer só em CI.
- **Stories US-101..US-110 marcadas `aprovado-retroativo`.** Foram entregues nas rc1/rc2 antes da regra `validate-story-approvals.js` existir. O audit trail está honestamente marcado como retroativo — não fingimos auditoria que não houve.
- **FISCAL-004 mudou texto.** Quem citou FS-DA como contingência em ADR antigo continua válido (FS-DA ainda existe), mas feature NOVA deve usar SVC. Não é breaking — só recomendação atualizada.

Nenhuma mudança breaking nesta release. Só adições e correções compatíveis com rc2.
