# v1.0.0-rc2

> **Segunda release candidate da v1.0.0.** Completa o port iniciado na rc1 — agora **TODO o framework + os 6 addons oficiais** rodam sem dependência de bash/perl/Git Bash. Se ninguém abrir issue grave em ~2 semanas, corta a v1.0.0 final.

## O que mudou

Os 5 últimos hooks bash que ainda viviam dentro dos addons oficiais foram portados pra Node.js puro. Com isso, **um cliente que rode `npx roldao-method add <addon>` em Windows sem Git Bash recebe um addon que funciona 100%** — antes alguns hooks de addon ficavam silenciosos nesse ambiente.

## Por que importa

- **Paridade entre core e addons** — depois da rc1 o core já rodava sem bash, mas addon ainda dependia. Quem usava `fintech-br`, `fiscal-br-completo`, `electron-br`, `esocial-completo` ou `varejo-pdv-br` em Windows puro tinha proteção parcial. Agora é integral.
- **Manutenção única** — todo hook do framework segue o mesmo contrato Node descrito em [ADR-013](../decisions/ADR-013-convencao-hook-node.md). Sem `.sh` espalhado em pasta de addon.
- **Suite de teste em `test/addons.test.js`** valida hooks `.js` via `node --check` + spawn — não depende mais de bash no runner CI dos addons.

## Como aplicar

```bash
npx roldao-method@latest update
```

Quem usa addon precisa rodar também, pra atualizar a versão `.js` do addon:

```bash
npx roldao-method add <addon>    # fintech-br, fiscal-br-completo, electron-br, esocial-completo, varejo-pdv-br
```

## Atenção

### Mudanças nos hooks de addon

| Addon | Hook antigo | Hook novo |
|---|---|---|
| `fintech-br` | `validate-webhook-signature.sh` | `validate-webhook-signature.js` |
| `fiscal-br-completo` | `require-sefaz-env.sh` | `require-sefaz-env.js` |
| `electron-br` | `block-ipc-without-validation.sh` | `block-ipc-without-validation.js` |
| `esocial-completo` | `validate-esocial-prazo.sh` | `validate-esocial-prazo.js` |
| `varejo-pdv-br` | `validate-tef-flow.sh` | `validate-tef-flow.js` |

Comportamento idêntico ao `.sh` original — só mudou a linguagem. Quem rodou `npx roldao-method add <addon>` antes da rc2 e tem instalação local com `.sh` desses addons: rode `add <addon>` de novo. O `settings.json.patch` foi atualizado pra apontar pra `node`.

### Sem breaking change pra core

- Hooks do core continuam exatamente os mesmos da rc1.
- `bash` e `perl` continuam não sendo requisito do framework.
- Quem está em v0.x e ainda não migrou: rode `update` e veja [`docs/MIGRACAO-V1.md`](../MIGRACAO-V1.md).

## Como testar

```bash
npm test                          # suite completa (11 fases, todas devem dar OK)
npm run test:addons               # smoke específico dos addons portados
npx roldao-method doctor          # confirma instalação local saudável
npx roldao-method add fintech-br  # confirma que addon instala com .js
```
